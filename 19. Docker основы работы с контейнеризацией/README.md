## Docker Homework — Custom Nginx

Кастомный образ nginx на базе Alpine с изменённой стартовой страницей.

### Образ в Docker Hub
https://hub.docker.com/r/timur0tonik/custom-nginx

### Сборка и запуск
```bash
docker build -t timur0tonik/custom-nginx:1.0 .
docker run -d -p 8080:80 --name my-nginx timur0tonik/custom-nginx:1.0
```

### Разница между образом и контейнером
Образ - это неизменяемый шаблон (набор слоёв файловой системы), созданный по Dockerfile. Он статичен, как "класс" в программировании.
Контейнер - это запущенный экземпляр образа: он включает образ + writable-слой + изолированный процесс в namespace. Контейнер динамичен, имеет жизненный цикл (start/run/stop/rm).
Один образ может породить сколько угодно независимых контейнеров.

### Можно ли в контейнере собрать ядро?
Технически - да, если запустить контейнер с --privileged, установить зависимости (build-essential, flex, bison, libssl-dev и др.) и смонтировать нужные хостовые пути (например, /lib/modules).

Однако это антипаттерн:
Контейнер использует ядро хоста и не может загрузить новое ядро:
- Сборка ядра - ресурсоёмкая задача, нарушает принципы контейнеризации ("один процесс - один контейнер"),
- Такой образ будет огромным и небезопасным.

Вывод: сборка возможна, но нецелесообразна. Для этого лучше использовать ВМ.
